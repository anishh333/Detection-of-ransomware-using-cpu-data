About RansomShield

Understanding the research methodology and system architecture

Research Paper

"Detection of Ransomware Attacks Using Processor and Disk Usage Data"

Authors: Kumar Thummapudi, Palden Lama, Rajendra V. Boppana

Published: IEEE Access, Volume 11, 2023 (pp. 51395–51407)

DOI: 10.1109/ACCESS.2023.3279819

Institution: Department of Computer Science, The University of Texas at San Antonio

Methodology

1

Data Collection

Monitor Hardware Performance Counters (HPCs) for 5 processor events and 8 disk I/O events from the host machine level. HPC data sampled at 5ms intervals; disk I/O at 20ms intervals.

2

Feature Engineering

Group 40 HPC samples into 200ms instances. Compute 4 statistical features (mean, median, std, IQR) per event. This yields 20 HPC features + 32 I/O features = 52 integrated features.

3

Model Training

Train with 5 data rounds using 4 ransomware samples (3 families) + 4 benign applications across 6 workload scenarios. 21-fold cross-validation.

4

Detection

The RF-Integrated model achieves 0.92 balanced accuracy and detects ransomware in the first 200ms instance 95% of the time. Within 400ms, detection probability is 98%.

HPC Events Monitored

HPC

LLC-stores

Last Level Cache store operations. Ransomware causes elevated cache activity due to intensive data processing during encryption.

HPC

L1-icache-load-misses

Level 1 instruction cache load misses. New code patterns from ransomware encryption routines cause increased misses.

HPC

branch-load-misses

Branch prediction load misses. Ransomware's encryption loops create unusual branch patterns.

HPC

node-load-misses

NUMA node load misses. Heavy data access patterns from file encryption cause cross-node memory access.

HPC

instructions

Total instructions executed. Fixed counter that captures overall processing activity level.

ML Classifiers

Best

Random Forest (RF)

Ensemble of decision trees. Best balanced accuracy (0.92) with lowest training time. Uses scikit-learn.

Support Vector Machine (SVM)

RBF kernel classifier. Good for high-dimensional feature spaces. Uses scikit-learn.

Decision Trees (DT)

Single tree classifier. Interpretable but prone to overfitting. Uses scikit-learn.

k-Nearest Neighbors (kNN)

Instance-based learning with k=5. Simple but computationally intensive at test time.

XGBoost

Extreme Gradient Boosting. Second-best accuracy but 3× training time vs RF. Uses xgboost library.

DNN (Deep Neural Network)

Auto-tuned with AutoKeras. Requires significant hyperparameter tuning.

LSTM

Long Short-Term Memory network with 64 units. Uses raw sequential data with masking layer.
